The Problem 01 / 08

Platforms Remember You. For Themselves.

ChatGPT, Claude.ai, Gemini all have memory now. Try exporting it. Best you get: raw transcripts. Your context is their moat.

Memory-as-a-Service Is Still Lock-in.

mem0 and supermemory add memory to your app. But data lives on their servers. They go down or raise prices -- your users lose everything.

Remembering Everything Isn't Memory. It's Hoarding.

RAG dumps everything into a vector store and hopes retrieval sorts it out. Real memory is knowing what matters, when to surface it, and what to forget. Filtering who, for what, and when -- that's the actual hard problem.

Switch Tools. Keep Everything.

Moved from Cursor to Claude Code? Your agent comes with you. New laptop? One import. Because with Signet, the knowledge was always yours.

Signet Is Not a Memory Service. It's a Standard.

We don't host your data. We don't run your queries. Signet defines how agent identity, memory, and config should be stored, searched, and carried between platforms. Hybrid retrieval that knows what matters now -- not just what's similar. Importance decay that lets your agent forget what doesn't serve it. YAML + Markdown in a directory you own. SQLite on your disk. An open spec anyone can implement.

Core Features 03 / 08

Curated Memory

View Spec →

Not a vector dump. Signet's memory system uses hybrid retrieval (semantic + keyword), importance decay, and a four-tier taxonomy (session, episodic, semantic, procedural) to surface what matters right now -- not everything that was ever stored. Your agent learns what to forget.

Switch from Cursor to Claude Code mid-project? Your agent remembers your architectural decisions, patterns, deploy setup -- and doesn't waste context on last month's debugging notes.

Secrets & Injection Safety

Learn More →

API keys encrypted at rest with libsodium. Secret values are never returned in model output -- even if a prompt injection attack tries to extract them. Agents use credentials through a secure proxy that redacts values from all output streams. The only agent framework with built-in prompt injection protection for secrets.

Your agent needs OPENAI_API_KEY to run evals and CLOUDFLARE_TOKEN to deploy. A malicious prompt tries to extract them. Signet's output redaction catches it before it leaves the sandbox.

Skills Registry

Browse Skills →

Install, vet, and manage agent skills from a curated registry. Skills are sandboxed -- they can access the capabilities they declare but nothing else. Review skill permissions before installation. No supply-chain surprises.

Your agent needs browser automation. signet skill install browser-use -- review its declared permissions, approve, done. The skill can browse but can't read your secrets or memory unless you grant it.

Multitenant Config

Architecture →

Manage one agent or hundreds from a single config. Each agent gets its own memory, identity, and secrets while sharing a common daemon and storage layer.

Personal coding agent + work research agent on same machine. Separate memory, secrets, personality. One CLI.

Quickstart 05 / 08
# Initialize signet in your home directory
> npm install -g signetai
Created ~/.agents/agent.yaml
Created ~/.agents/memory.db
# Store a memory (auto-embedded)
> signet remember "prefer bun over npm" -t preference
[ok] Memory stored (importance: 0.72)
# Query memories
> signet recall "package manager preferences"
[1] prefer bun over npm (0.72)
# Manage secrets
> signet secret set OPENAI_API_KEY
[ok] Secret encrypted and stored
# Check status
> signet status
Agent: mr-claude | Memories: 847 | Secrets: 12 | Daemon: running
// @signet/sdk — use from any Node.js app
import { Signet } from '@signet/sdk';
const agent = new Signet();
// Store a memory
await agent.remember('migrated to bun', {
tags: ['build', 'decision'],
importance: 0.8
});
// Recall with hybrid search
const results = await agent.recall('build tools');
// Access secrets (value never in model output)
const key = await agent.secret('OPENAI_API_KEY');
# ~/.agents/agent.yaml
name: mr-claude
version: "0.2.1"
personality: ./soul.md
memory: ./memory.db
harnesses:
claude-code:
output: ~/.claude/CLAUDE.md
cursor:
output: ./.cursorrules
opencode:
output: ./AGENTS.md
memory_backend:
embedding: nomic-embed-text
search: hybrid
decay: true
