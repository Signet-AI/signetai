# Doc Drift Fix — Automated Agent Prompt

You are a documentation maintenance agent. Your job is to detect documentation
drift in this repository and open a PR to fix it.

## Step 1: Run the drift detector

```bash
bun scripts/doc-drift.ts --markdown
```

If exit code is 0, there is no drift. Reply "No documentation drift detected."
and stop.

If exit code is 1, continue to Step 2.

## Step 2: Create a branch

```bash
git checkout -b docs/drift-fix-$(date +%Y-%m-%dT%H-%M-%S)
```

## Step 3: Fix each category of drift

Capture the current drift state before making any edits — this will be used
for the commit message and PR description:

```bash
DRIFT_JSON="$(bun scripts/doc-drift.ts 2>/dev/null)"
DRIFT_REPORT="$(bun scripts/doc-drift.ts --markdown 2>/dev/null)"
DRIFT_BULLETS="$(printf '%s' "${DRIFT_JSON}" | jq -r '.summary[]' 2>/dev/null | sed 's/^/- /')"
```

Use `${DRIFT_JSON}` for structured data (routes, migrations, packages)
throughout Step 3 — do not re-run the script again for JSON output.

### Route drift (missing from CLAUDE.md)

For each route missing from CLAUDE.md, add a row to the HTTP API Endpoints
table. The table format is:

```
| `/path` | METHOD | Description |
```

- Match the existing table style exactly (backtick-wrapped paths, pipe-separated)
- Group new routes logically near related existing routes (e.g., marketplace
  routes together, 1password routes near secrets)
- For the description column, read the route handler in the source file to
  understand what it does and write a concise description
- If multiple methods serve the same path, combine them: `GET/POST`
- Do NOT remove existing rows unless the drift report says they're extra

### Route drift (extra in CLAUDE.md)

If routes appear in CLAUDE.md but not in source, remove those table rows.
Double-check the source files first — the route may have been renamed.

### Migration drift

Update the migration range strings in CLAUDE.md. There are typically two
locations:

1. The prose description under "Database Migrations" (e.g., "001-baseline
   through 010-umap-cache") — update the end to match the actual latest file
2. The Key Files bullet (e.g., "Database migrations (001 through 012)") —
   update the number to match the actual latest

Get the actual latest from the drift report's `migrations.actualMax` field.

### Key files drift

If any paths listed in the Key Files section no longer exist on disk, remove
them from the list.

### Package drift

For packages missing from the CLAUDE.md table, add a row. Read the package's
`package.json` to get the name and description. Use the existing table format:

```
| `package-name` | Description | target |
```

For the README.md packages table, the format is:

```
| [`package-name`](./relative/path) | Role description |
```

Fix any name mismatches (e.g., `@signetai/adapter-openclaw` should be
`@signetai/signet-memory-openclaw` if that's the actual package.json name).

Remove table entries for packages that don't exist on disk.

## Step 4: Verify the fix

Run the drift detector again:

```bash
bun scripts/doc-drift.ts --markdown
```

It should report no drift (exit 0). If drift remains, fix it and re-check.

## Step 5: Commit and PR

```bash
git add CLAUDE.md README.md docs/
git commit -m "docs: fix documentation drift

${DRIFT_BULLETS}"

PR_BODY_FILE="$(mktemp)"
cat > "${PR_BODY_FILE}" <<'PREOF'
## Summary

Automated documentation drift fix. Changes detected by `scripts/doc-drift.ts`:

PREOF
# Skip the first 4 lines of the report (title, blank, ## Summary, blank)
# to avoid an h1 nested under the PR body's h2 and a duplicate ## Summary.
printf '%s\n' "${DRIFT_REPORT}" | tail -n +5 >> "${PR_BODY_FILE}"
cat >> "${PR_BODY_FILE}" <<'PREOF'

---
*Generated by doc-drift scheduled task*
PREOF

gh pr create \
  --title "docs: fix documentation drift" \
  --body-file "${PR_BODY_FILE}"
rm -f "${PR_BODY_FILE}"
```

## Rules

- Only modify documentation files (CLAUDE.md, README.md, docs/*.md)
- Never modify source code
- Match existing formatting exactly — no style changes
- If unsure about a description for a new route, read the handler code
- Keep changes minimal and focused on drift fixes only
